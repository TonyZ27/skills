# Vibe Coding Recap

## Session Goal
State what the user was trying to accomplish. If the goal shifted mid-session, name the original goal and the revised goal.

## What Actually Happened
Summarize the session arc in a few beats. Focus on turning points rather than every step.

## What Worked Well
Call out the prompts, habits, decisions, or collaboration patterns that clearly helped.

## Where Friction Showed Up
Identify the moments that caused churn, confusion, rework, or drift.

## Patterns Behind The Session
Extract the deeper patterns. These can be about scoping, prompting, verification, debugging style, switching costs, or decision timing.

## Lessons Worth Keeping
Name the repeatable moves the user should reuse.

## What To Change Next Time
Give concrete, behavioral advice. Prefer 3-5 actions that are specific enough to apply in the next session.

## One-Sentence Coaching Takeaway
End with a concise line the user could remember going into the next vibe coding session.
